# Hackathon Submission

Use when packaging research for Track 1 and Track 2.

## Track 1: Research Article

Output:

- Title
- One-sentence thesis
- 5-7 section outline
- Findings grid summary
- Caveats
- Source list
- X post draft

Quality bar:

- Clear position
- Recent market move
- Evidence-backed explanation
- Original angle
- Honest uncertainty

## Track 2: Research Agent Or Workflow

Output:

- What the skill does
- Why it is useful
- How it works
- Repository structure
- One live example
- Demo prompt
- Expected output

Quality bar:

- The workflow must be reusable beyond one article.
- The live example must show decomposition, source mapping, triangulation, and synthesis.
- The demo must include at least one caveat that weakens the thesis.

## Live Demo Script

1. Show the messy question.
2. Run intake.
3. Decompose into sub-questions.
4. Assign source categories.
5. Triangulate the highest-stakes claim.
6. Fill the findings grid.
7. Generate the thesis and article outline.
8. Show how the same result becomes the Track 1 article.

## Submission Checklist

- README explains problem and usage
- `skill/SKILL.md` has clear frontmatter
- Modules are linked from the entry point
- Installers are non-destructive
- Validator passes
- Demo prompt is included
- No fabricated metrics or claims
